The Input Reality

We start with 133 Keypoints per frame.

  • Video Rate: Sampled at 25 FPS.
  • Issue: Fast vs Slow signers produce different sequence lengths for the same sign.

Phase 2: Q-grid Sampling

Clocking by Motion

We discard the video timestamp. We only record a new frame when a joint moves a cumulative 10 pixels.

Result: "Time Dilation"

Fast and slow signers producing the same spatial shape yield sequences of the exact same length.

Deep Dive 1: Token to Vector

1. Embedding

Input: Token 5. Output: Vector size 8.

[1.0, 1.0, 0.2, -0.1, 0.05, 0.9, -0.3, 0.1]

First 2 numbers are hard-coded physics (X= +1, Y= +1). The rest are learned.

2. Flattening

We combine 133 joints into one massive vector per frame.

Calculation: 133 Joints × 8 Dims = 1,064 Dims.

This represents the entire body posture at one millisecond.

Deep Dive 2: Project & Compress

1. Projection

Goal: Reduce 1,064 noisy dimensions to a dense feature set.


Input: (Batch, T, 1064)
Layer: Linear(1064, 512)
Output: (Batch, T, 512)

Result: A "Pose Summary" vector of size 512 for every time step.

2. Compression

Goal: Squeeze Time (T) to handle long sequences.


Config: compress_factor = 4
Op: Conv1d(Stride=4)
Input T: 12,000 Frames
Output T: 3,000 Frames

Result: Removes micro-jitters and expands the "receptive field" for Mamba.
